# JRPG Island

A small top-down JRPG-style island prototype built in **Godot 4.6** (Compatibility
renderer, 640x480 viewport, pixel-art at 16px tiles). Walk around a circular
island, talk to an NPC, read signposts, and open a chest — the beginnings of an
interactive JRPG.

## Running the game

1. Open the project in Godot 4.6+ (`project.godot`).
2. The main scene is `scenes/main.tscn` — press **Play** (F5).

## Controls

| Action            | Keys                      |
|-------------------|---------------------------|
| Move              | `W` `A` `S` `D` or Arrows |
| Interact / Talk   | `E`, `Space`, or `Enter`  |

Movement is tile-by-tile (grid-snapped). You can't walk onto water tiles.

## Interacting

Walk up to an interactable and face it, then press **Interact**. Interactables on
the island:

- **Elder** (NPC) — gives some lore and a hint about the chest.
- **Chest** — awards an item ("Herb of Tides").
- **Signpost** — world flavor text.

Dialogue uses a typewriter effect; press **Interact** to fast-forward a line, and
again to advance to the next line. Movement is disabled while a dialogue box is open.

## Rebuilding the island

The island scene is generated by a headless build script (terrain rings, Wang-tile
transitions, prop placement, interactables). Re-run it any time you change the
island layout logic in `scripts/build_island.gd`:

```bash
godot --headless --script scripts/build_island.gd
```

This regenerates `assets/tilesets/island.tres` and `scenes/main.tscn`.

### Rendering a screenshot

A overview render can be produced without the editor:

```bash
# Headless mode has no viewport rendering; use a virtual display:
xvfb-run -a -s "-screen 0 640x480x24" godot res://scenes/render_test.tscn
# Output is written to render_output.png
```

## Project layout

```
scenes/
  main.tscn          # The game scene (terrain + props + player + UI)
  player.tscn        # Player CharacterBody2D (sprite + camera)
  dialogue_box.tscn  # Dialogue UI (panel + name + body + hint)
  render_test.tscn   # Headless overview-render scene
scripts/
  main.gd            # Scene root: sets up layers + DialogueBox
  player.gd          # Grid movement + facing-aware interaction
  interactable.gd    # Base script for talkable/examinable props
  dialogue_box.gd    # Typewriter dialogue UI controller
  global.gd          # Autoload: tile size + map/world helpers
  build_island.gd    # Headless island generator (see above)
  render_scene.gd    # Screenshot helper used by render_test.tscn
assets/
  sprites/           # Characters, NPCs, buildings, items, props (PNG)
  tilesets/          # island.tres + generated transition atlases
project.godot        # Input maps (move_* + interact), renderer/display config
```

## How interactions work

Any `Sprite2D` becomes interactable by attaching `scripts/interactable.gd` and
setting two metadata entries:

- `display_name` (string) — shown as the speaker name.
- `dialogue` (array of strings) — lines shown in order.

The player (`scripts/player.gd`) listens for the `interact` action, computes the
tile directly in front of its facing direction, and scans its siblings for an
interactable on that tile. On a match it calls the interactable and opens the
`DialogueBox`. The build script wires up the Elder, Chest, and Signpost this way
in `place_npc()` / `place_interactable()`.

## Notes

- Assets are loaded at runtime with `Image.load_from_file()` (not as pre-imported
  textures), so the Godot `.import`/`.uid` artifacts are not committed.
- The `pixellab` MCP (`.opencode/mcp.json`) was used to generate the pixel-art
  sprites and tileset transitions; that config is gitignored because it holds an
  API key.
